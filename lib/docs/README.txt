Кейсы ARS:

Сервисы:

*) При отсутствии headless, создание базового UI. Сворачивание UI при шатдауне.
*) При сетапе PROBE шедулера. При шатдауне остановить симуляцию. При автостарте
   стартануть симуляцию. При наличии UI, зарегистрировать панель управления
   симуляцией.
*) Старт-стоп терминала.

Приложения:
*) Создать объекты данных. Запланировать задачи через шедулер.
*) При наличии UI создать элементы интерфейса при запуске. Остановить обработку
   данных обработчиками элементов управления при останове.

Что такое выходной сигнал?
Выходной сигнал это объект блокировки текущего потока, позволяющий дождаться
завершения application runtime service.

Кому нужен выходной сигнал?
*) Выходной сигнал нужен только основному потоку, для отслеживания когда нужно
завершить работу приложения.
